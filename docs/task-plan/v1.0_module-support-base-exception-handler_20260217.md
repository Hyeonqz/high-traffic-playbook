# v1.0 — module-support BaseGlobalExceptionHandler 작업 계획

## 작업 개요
module-support 에 `BaseGlobalExceptionHandler` 를 추가하고,
case-ticketing 의 `GlobalExceptionHandler` 가 이를 상속하도록 리팩토링한다.

## 작업 대상
- `module-support`
- `case-ticketing`

---

## 작업 순서

### Step 1. module-support — ErrorResponse 이동
- `io.github.hyeonqz.exception.ErrorResponse` 를 module-support 에 생성
- 현재 case-ticketing `GlobalExceptionHandler.kt` 내부에 정의된 `ErrorResponse` 를 제거

### Step 2. module-support — BaseGlobalExceptionHandler 구현
- 패키지: `io.github.hyeonqz.exception`
- 처리 대상 예외:
  | 예외 | HTTP Status | 코드 |
  |------|-------------|------|
  | IllegalArgumentException | 400 | INVALID_ARGUMENT |
  | NullPointerException     | 500 | NULL_POINTER     |
  | RuntimeException         | 500 | RUNTIME_ERROR    |
  | Exception                | 500 | INTERNAL_ERROR   |

### Step 3. case-ticketing — GlobalExceptionHandler 상속 적용
- `GlobalExceptionHandler : BaseGlobalExceptionHandler()` 로 변경
- 도메인 고유 예외만 유지:
  - `TicketingException`
  - `ObjectOptimisticLockingFailureException`
- 제거: 기존 generic `Exception` 핸들러 (Base 로 위임)

### Step 4. 문서 작성
- `.claude/docs/feature/module-support-v1.0-base-exception-handler.md`

---

## 영향 범위
- 수정 파일: `case-ticketing/.../exception/GlobalExceptionHandler.kt`
- 신규 파일: `module-support/.../exception/ErrorResponse.kt`
- 신규 파일: `module-support/.../exception/BaseGlobalExceptionHandler.kt`
- 의존성 변경: 없음 (case-ticketing 이미 module-support 의존)
